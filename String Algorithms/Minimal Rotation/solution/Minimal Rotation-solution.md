# Minimal Rotation - Solution

**ID:** 1110 | **URL:** https://cses.fi/problemset/task/1110

## Problem Statement

Given a string $s$ of length $n$ consisting of characters 'a'â€“'z', a rotation is generated by moving characters one after another from the beginning to the end. Print the lexicographically smallest rotation.

## Approach

### Step 1: Reducing by doubling the string

Concatenate $s$ with itself to get $t = s + s$ of length $2n$. Every rotation of $s$ is exactly a substring of length $n$ of $t$ starting at position $1, 2, \ldots, n$. The problem reduces to: find the index $i \in [1, n]$ such that $t[i..i+n-1]$ is lexicographically smallest.

### Step 2: Booth's algorithm

Maintain $i$ as the best candidate (starting position) found so far, and compare it successively with candidates $c > i$, eliminating many worse positions at once via domination. Let $d$ be the length of the common prefix already matched between $t[i..]$ and $t[c..]$; compare $t[i+d]$ with $t[c+d]$:

- $t[i+d] < t[c+d]$: $c$ is larger than $i$ at offset $d$; candidates $c, c+1, \ldots, c+d$ are dominated, skip the whole range and move to $c+d+1$ (reset $d=0$).
- $t[i+d] = t[c+d]$: extend the matched prefix, $d \gets d+1$.
- $t[i+d] > t[c+d]$: $i$ is larger than $c$, $i$ loses; candidates $i+1, \ldots, i+d$ are dominated, so move $i$ to $c$ and continue (repeat the jump if $c$ still lies inside the dominated range).

**Domination property (ensures the optimum is not skipped):** If $t[i..i+d-1] = t[c..c+d-1]$ and they first differ at offset $d$, then for each $r \in [0, d]$, candidates $i+r$ and $c+r$ match for $d-r$ characters and then also differ at the position corresponding to offset $d$ (since $t[i+d] \ne t[c+d]$). Hence:
- when $t[i+d] < t[c+d]$: $c+r > i+r$ for all $r \in [0,d]$, so $c, c+1, \ldots, c+d$ are not the smallest rotation;
- when $t[i+d] > t[c+d]$: $i+r > c+r$ for all $r \in [0,d]$, so $i+1, \ldots, i+d$ are not optimal, and $c$ is better than $i$.

Thus every skipped range certainly does not contain the answer; the algorithm never misses the smallest rotation. (The accompanying code performs this in place with pointers $k = i+d$ and $j = c+d$, comparing $S[k]$ with $S[j]$.)

### Example

Using the example from the problem statement: $s = \text{"acab"}$ ($n = 4$), $t = \text{"acabacab"}$.

The rotations (substrings $t[i..i+3]$ starting at $i = 1..4$) are: `acab`, `caba`, `abac`, `baca`.

Booth scan (comparing candidate $i$ with candidate $c$, $d$ = matched length):

- $i=1$ (`acab`) vs $c=2$ (`caba`): $t[1]=\text{a} < t[2]=\text{c}$ $\Rightarrow$ candidate 2 is larger, skip.
- $i=1$ (`acab`) vs $c=3$ (`abac`): match $t[1]=t[3]=\text{a}$ (1 char), then $t[2]=\text{c} > t[4]=\text{b}$ $\Rightarrow$ $i$ loses, jump $i \to 3$ (candidate 2 is dominated, skip).
- $i=3$ (`abac`) vs $c=4$ (`baca`): $t[3]=\text{a} < t[4]=\text{b}$ $\Rightarrow$ candidate 4 is larger, skip.

The remaining candidates are either larger or outside the rotation range $[1, n]$, so $i=3$ is optimal. Answer: start at $i=3$, print $t[3..6] = \text{"abac"}$.

Result:
```
abac
```

## Complexity

- **Time:** $O(n)$ $\textemdash$ each character comparison either advances the scanning pointer, and each time $i$ loses, $i$ jumps forward; the total number of comparisons is at most $2n$ (the doubled length), plus $O(n)$ index jumps.
- **Memory:** $O(n)$ $\textemdash$ storing the doubled string $t$ of length $2n$.
